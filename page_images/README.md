# Page Images - KindleModShelf Screenshot System

This folder contains screenshots and images for tool/mod pages on KindleModShelf.

## Image Naming Convention

Images should be named following this pattern:
```
[page-name]-[number].[extension]
```

### Examples:
- `koreader-1.jpg` - First screenshot for koreader.html
- `koreader-2.jpg` - Second screenshot for koreader.html
- `2048-1.png` - First screenshot for 2048.html game
- `kpaint-1.jpg`, `kpaint-2.jpg`, `kpaint-3.jpg` - Multiple screenshots for kpaint.html

### Guidelines:
- Use lowercase for page names
- Use hyphens to separate words (e.g., `kindleforge-1.jpg`)
- Number images sequentially starting from 1
- Use `.jpg` for photographs and complex screenshots
- Use `.png` for screenshots with text, UI elements, or transparency
- Keep file sizes reasonable (under 500KB when possible)

## Image Dimensions

Recommended dimensions:
- **Standard screenshots**: 800-1200px wide
- **Feature images**: 1600px wide
- **Thumbnails**: Auto-generated by browser

Aspect ratios supported:
- `3:2` (default) - General screenshots
- `16:9` (wide) - Widescreen screenshots
- `1:1` (square) - Icons or square content
- `3:4` (portrait) - Portrait screenshots

## Required Files

To use the screenshot system, pages must include:

1. **style.css** (already included in all pages)
2. **screenshots.js** - Add before closing `</body>` tag:
   ```html
   <script src="screenshots.js"></script>
   ```

## Features

The screenshot system includes:
- ✅ Click to open lightbox viewer
- ✅ Keyboard navigation (← → arrows, ESC to close)
- ✅ Smooth animations and hover effects
- ✅ Lazy loading support
- ✅ Dark/light theme compatibility
- ✅ Fully responsive (mobile-friendly)
- ✅ Multiple gallery support on one page
- ✅ Image counter (e.g., "Screenshot 2 / 5")

## Browser Support

- Modern browsers (Chrome, Firefox, Safari, Edge)
- Kindle browser (graceful degradation)
- Mobile browsers (iOS Safari, Chrome Mobile)

## Tips

1. **Optimize images** before uploading - use tools like TinyPNG or ImageOptim
2. **Use descriptive alt text** for accessibility
3. **Test on Kindle browser** - screenshots should show what it actually looks like
4. **Group related screenshots** in the same gallery
5. **Use captions** to explain what the screenshot shows
6. **Consider using lazy loading** for pages with many images
